A complete architectural and product blueprint for a modern, SEO-first, scalable editorial platform. Use this as your single source of truth from discovery through launch.

A. Product Overview
Vision: Build a premium, editorial-grade blogging platform that rivals Medium, Ghost, and Substack in reading experience, while giving content teams the power of a headless CMS (Strapi/Sanity-class) with built-in SEO intelligence, performance budgets, and growth tooling.

Positioning: Not "a blog" — a content operating system. Three surfaces:

Reader-facing site — fast, beautiful, accessible, SEO-dominant.
Admin/CMS — structured content, workflows, roles, audit trails.
Author workspace — block editor with live preview, AI assist, and SEO scoring.
Differentiators:

Block-based editor with live multi-device preview and SEO score gating publish.
Built-in topic clusters & internal-link suggester (semantic).
Auto-generated OG images, schema, and sitemaps.
Core Web Vitals budgets enforced in CI.
First-class dark mode, reading progress, bookmarks, reactions.
Newsletter + membership-ready from day one.

I. SEO and Technical SEO System
On-page

One H1 per page; auto-derived from title; H2/H3 enforced hierarchy.
Editable slug with kebab-case validator, history of old slugs → auto 301.
Meta title/description with templates (%title% | %site%), per-route overrides.
Canonical tag on every page; default self-canonical; cross-domain support.
Open Graph + Twitter cards; per-post OG image with auto-generated fallback (Satori).
Internal-link suggester (semantic + same-cluster).
Topic clusters: pillar pages link to cluster posts and vice versa.
Schema (JSON-LD)

Organization, WebSite (+ SearchAction), BreadcrumbList, Article/BlogPosting, Person (author), FAQPage, HowTo, ImageObject, VideoObject. Validate via schema.org and Rich Results Test in CI.
Crawlability

robots.txt generated from settings; environment-aware (block noindex env).
sitemap.xml index + segmented (posts, categories, tags, authors, pages); only canonical, 200, indexable URLs; lastmod from DB.
rel=next/prev on paginated archives + canonical to page.
Hreflang for multilingual.
Hygiene

301/302 manager, redirect-chain detector (>1 hop = warn).
Orphan-page report, duplicate-content detector (shingles).
Broken-link crawler (weekly).
HTTPS only, HSTS, single canonical host (apex vs www).
Image SEO: required alt, descriptive filenames, lazy-loading except LCP.
Pre-publish SEO checklist (gate publish): title 30–60 chars, desc 120–160, focus keyphrase in title/H1/first 100 words, slug ≤ 60 chars, ≥1 image with alt, ≥2 internal links, schema valid, OG image present.

J. Performance Optimization System
Targets: LCP < 2.5s, CLS < 0.1, INP < 200ms (mobile p75).

Rendering

SSG for posts/listings + ISR (revalidate on publish via webhook).
Edge SSR for personalized surfaces (bookmarks, auth).
Streaming + Suspense for above-the-fold priority.
Assets

AVIF/WebP via image pipeline; responsive srcset + sizes; LCP image priority.
font-display: swap, self-hosted variable fonts, subset by language.
Code-split routes; defer non-critical JS; tree-shake; no client JS on static blocks.
Preload hero image + critical font; preconnect to CDN/analytics.
Caching

CDN (Cloudflare/Vercel Edge) with stale-while-revalidate.
HTTP cache headers: long-immutable for hashed assets, SWR for HTML.
Redis for hot queries (post by slug, sitemap fragments, trending).
Postgres: indexes on slug, status+published_at, author_id, category_id, GIN on tags/tsvector.
Monitoring

Web Vitals RUM → store in DB; dashboard widget.
Lighthouse CI on PRs; budgets fail build (JS ≤ 170KB gz, image LCP ≤ 200KB).
Sentry performance traces.
K. UI/UX and Design Direction
Aesthetic: editorial premium — think The Verge × Stripe Press × Linear. Confident typography, generous whitespace, restrained color, purposeful motion.

Design tokens (CSS variables, oklch)

Color: --bg, --surface, --surface-elev, --fg, --fg-muted, --border, --primary, --primary-fg, --accent, --success, --warn, --danger. Light + dark.
Typography: display (e.g., Fraunces or Instrument Serif), body (Inter or Geist), mono (JetBrains Mono). Modular scale 1.250.
Spacing: 4px base, scale 0.5–24.
Radii: sm 6, md 10, lg 14, xl 20, 2xl 28.
Shadows: layered, low-opacity.
Motion: 150–300ms, ease-out; respect prefers-reduced-motion.
Components: Button, Input, Select, Combobox, Dialog, Sheet, Tabs, Tooltip, Toast, Card, Avatar, Badge, Breadcrumb, Pagination, Skeleton, EmptyState, DataTable, Editor blocks. All keyboard-accessible (WCAG 2.2 AA).

S. Content Workflow
Author creates draft → autosave → submits for review.
Reviewer/Editor receives notification → comments inline → requests changes or approves.
SEO Manager validates meta, schema, internal links → SEO score must pass threshold.
Editor schedules or publishes; webhook triggers ISR revalidation + sitemap regen + search index update.
Revisions stored on every save; restorable; diff viewer.
Slug change → old slug auto-added to redirect table (301).
Update workflow: "Refresh" badge after 180d; updates increment updated_at and lastmod.
Rollback: one-click restore to revision N.
Preview: signed URL valid 24h, shareable with non-logged-in stakeholders.
T. Analytics and Reporting
Custom RUM table page_views partitioned monthly; aggregated nightly into post_metrics_daily.
Sources: direct, organic, referral, social, email, paid (UTM parsed).
Reports: top posts, top authors, top categories, top entry pages, exit pages, internal search queries (and zero-result queries), scroll depth (25/50/75/100), reading completion (time-on-page vs reading time), CTA funnel (view → click → convert), newsletter funnel.
Content decay: posts with 30d traffic ↓ >40% vs prior 30d.
Refresh queue: published >180d AND (impressions high OR ranking 8–20).
Export CSV + scheduled email digests.
U. Security and Reliability
Auth: bcrypt/argon2 hashes, secure cookies (httpOnly, SameSite=Lax), session rotation, optional TOTP 2FA, password HIBP check.
RBAC server-side on every mutation; never trust client claims.
RLS policies on all user-data tables; has_role() SECURITY DEFINER.
Input validation with zod on every endpoint (min/max/regex).
XSS: sanitize rendered HTML (DOMPurify on server) for any user-submitted markup; sandboxed iframes for embeds.
CSRF: same-site cookies + token on state-changing POSTs from browser.
Rate limiting (Redis token bucket): auth, comments, search, AI.
File uploads: MIME sniff, size cap, EXIF strip, magic-byte check, virus scan (ClamAV optional).
Spam: Cloudflare Turnstile on forms + comments; Akismet-style heuristic.
Secrets in env (server-only process.env), never VITE_*.
HTTPS + HSTS + CSP (script-src nonce), Referrer-Policy, Permissions-Policy.
Webhook signature verification (HMAC + timing-safe compare).
Backups: nightly Postgres + media; PITR; quarterly restore drill.
Monitoring: Sentry, uptime checks, log aggregation, alerting.
Audit log immutable (append-only); diff stored.
